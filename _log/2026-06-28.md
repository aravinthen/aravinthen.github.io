## Notes
### RL experiments
1. Oh, crap. I figured out what (part) of the issue is.
2. I've been handling truncation and termination as essentially the same case. This has consequences for how the value propagates throughout the calculation: a terminal case should have a value of zero, as the expected *future* return of a terminal case should be zero. However, this is *not* the same as truncation - truncation occurs when a run is terminated by external conditions. This is precisely what is happening in the `pendulum` environment!
3. Fixing this demonstrates a noticeable improvement in the learning signal. But not much...
4. I'm realising that my issues here arise from how I've essentially transplanted much of my code from my vanilla actor-critic implementation into the new advantage actor-critic implementation. I have a feeling I'm going to have to put a bit more thought into what it is that the actor generates with respect to the new environment...
